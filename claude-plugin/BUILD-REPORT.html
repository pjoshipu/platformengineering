A Claude Code plugin that mirrors this repo's web Admin Console (the Agentic AI in Platform Engineering app) as native slash commands.
Claude Code plugin 9 slash commands 2 skills 8 prompt templates 5 personas Build report · 2026-06-18

1. What this is & why

The repository platformengineering is a React/Vite course-demo app. Its web UI has an Admin view with 8 AI-agent "tabs" and a Developer view with a copilot. Today each agent is a Supabase Edge Function (Deno/TS) that builds a prompt and calls an external LLM (gpt-4o-mini / claude-3-haiku).

This plugin re-implements those tabs as Claude Code slash commands. Because Claude Code is itself the model, the agents run natively in the conversation — reusing the exact same prompts — with no API keys, no servers, and no network calls. Typing /admin opens the console, /on-board interactively asks the new hire's persona before tailoring a plan, and every other tab has an equivalent command.

Decisions confirmed with the user before building:
  • Execution: Claude runs the agents natively (no external backends).
  • Scope: all 8 agents + an /admin launcher.
  • Location: in-repo, so prompts stay version-controlled next to the edge functions they mirror.
  • Roles: Admin scope for now (the developer copilot still ships as one of the 8 agents).

3. Architecture — three primitives, one job each

PrimitiveRoleWhy
commands/*.md Thin entry points = the "tabs". Gather inputs (from $ARGUMENTS or via AskUserQuestion) and point Claude at the right template. No duplicated prompt text — the heavy prompt lives in exactly one place.
skill: pe-prompts One canonical prompt template per agent, lifted verbatim from supabase/functions/<agent>/index.ts. Single source of truth; behavior matches the web app and never drifts.
skill: pe-personas Persona reference files read by /on-board to tailor onboarding by role. Richer than the web app, which only had free-text team/background.

Subagents were intentionally left out of v1 — native inline execution keeps the interactive /on-board flow clean. Bundled files are referenced via ${CLAUDE_PLUGIN_ROOT} so they resolve regardless of install location.

6. The interactive /on-board flow

Step 1 — Gather inputs. Parse $ARGUMENTS; for anything missing, call AskUserQuestion for Role (Platform / Android / Frontend / Backend dev, + Other → Data Engineer), Level (Junior 14d / Intermediate 7d / Senior 5d), Team, and Learning style (Async / Pair / Mixed).
Step 2 — Load persona context. Map the chosen role to skills/pe-personas/personas/<role>.md and read it.
Step 3 — Run the agent prompt. Read skills/pe-prompts/templates/developer-onboarding.md, fill placeholders, and apply the experience-based ramp plus the persona's ramp-time modifier.
Step 4 — Output. A formatted plan (checklist by category, ranked resources, first-week agenda, success metrics, mentor recommendation, ramp estimate); raw JSON on request.

Sample command file — commands/on-board.md (frontmatter)

---
description: Developer Onboarding agent — interactive, persona-tailored onboarding plan for a new hire.
argument-hint: [name] [team] [junior|intermediate|senior]
allowed-tools: Read, AskUserQuestion
---

8. Install & enable

From the repo root in a Claude Code session:

/plugin marketplace add ./claude-plugin
/plugin install platform-console@platform-eng
/reload-plugins

To auto-enable for the whole team on clone, commit this to the repo's .claude/settings.json:

{
  "extraKnownMarketplaces": {
    "platform-eng": {
      "source": { "source": "directory", "path": "./claude-plugin" }
    }
  },
  "enabledPlugins": {
    "platform-console@platform-eng": true
  }
}

The enabledPlugins object-map shape (not an array) was verified against the current Claude Code plugin-marketplace docs.
